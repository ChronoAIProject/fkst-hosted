Add `POST /api/v1/repos/:owner/:name/fkst-setup` (#181): an idempotent endpoint that initializes an existing GitHub repo for fkst by committing a `.fkst/` directory onto the repo's resolved default branch in a single atomic commit (`fkst: initialize fkst context`). It writes a minimal valid example package (`.fkst/packages/example/departments/example/main.lua` + `README.md`, conformant with the engine's package validation) and a default per-repo `.fkst/AGENTS.md` base instruction file. The commit goes through the GitHub **App installation token** (the only credential holding `contents:write`), via a new `github_hub::contents::commit_files` Git Data API writer (blobs → tree → commit → ref, reusing the base tree so existing content is preserved). Gated by a dedicated `fkst:repo:setup` permission (separately grantable from `fkst:goal:create` — least privilege; NyxID must assign it) plus an optional `org_id` org-writer check and the App installation probe. Idempotent and non-destructive by default: an existing `.fkst` returns `200 already_initialized` with no write; `?force=true` re-commits only the three scaffold paths. The App token and scaffold contents are never logged.
