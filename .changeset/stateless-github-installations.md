Make GitHub App installation resolution **stateless** (issue #141): remove the `github_installations` Mongo collection, `MongoInstallationStore`, the `InstallationStore`/`StoredInstallation` seam, and the `GithubInstallationDoc` model. Installations are now resolved on demand via `GET /repos/{owner}/{repo}/installation` and cached in memory with a jittered TTL (`INSTALLATION_TTL_BASE` 15 min + up to `INSTALLATION_TTL_JITTER` 5 min, stored as a per-entry `expires_at`) so a worker fleet does not re-probe in lockstep; a stale installation self-corrects at the next mint via the existing `InstallationGone` path. The App webhook keeps its signature verification, its always-2xx redelivery-safe policy, its cache eviction, and its "fail affected live sessions" behaviour, but drops ALL durable reads/writes — it is now a pure cache-bust hint (in-memory eviction per affected repo, or account-wide when the event enumerates none) that fans the eviction to other workers through an injectable `InstallationEvictionBroadcaster` seam (a `NoopEvictionBroadcaster` default until the controller→worker evict channel is wired in #134/#151). `db.rs` drops the installation collection/indexes/accessor and `main.rs` constructs `GithubAppTokens::new(&config)` (no store). Documented in `docs/github-app.md`, including the N-worker cold-start REST rate-limit note.
