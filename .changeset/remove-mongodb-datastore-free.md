Remove the MongoDB dependency and make the controller datastore-free (completes #143).

This is the residual Mongo deletion after the session store went in-memory (#198-i) and the lease/distribution layer was deleted (#198-ii). All remaining Mongo is now gone: the `Db` handle (`src/db.rs`), the `AppError::Mongo` variant, the `MONGODB_*` configuration (`mongodb_uri`/`mongodb_db`/`mongodb_server_selection_timeout_ms`), the `AppState.db` field, and the `/health` datastore ping. `/health` and `/api/v1/health` now report ready unconditionally (`{"status":"ok","version":"<version>"}`, no `mongo` field) — a controller that can answer the route is healthy. The unprefixed platform secrets (`GITHUB_TOKEN` / `NYXID_CLIENT_ID` / `NYXID_CLIENT_SECRET`) and the internal worker-protocol knobs that previously rode the unprefixed `MongoVars` envy pass survive in a renamed `PlatformVars` pass; no `MONGODB_*` variable is read or required at startup. The dead `LeaseDoc` model is deleted (the `SessionDoc` bson shape — used by the in-memory `SessionRepo` CAS — is unchanged). The `mongodb`, `testcontainers`, and `testcontainers-modules` crates are dropped from the dependency graph (`bson` stays for `SessionDoc`'s value types + the CAS round-trip; `wiremock` stays for the fake-GitHub tests). The integration suite was rewritten to run with NO Mongo container and NO Docker: every harness that built a testcontainer Mongo now builds `AppState` with no datastore and runs unconditionally; `tests/persistence.rs` (Mongo index/document persistence) is deleted.
