Session-lifecycle goal-issue labels + persisted terminal cause (#180). Replace the goal-status `status:*` label scheme on a goal's GitHub Issue with a session-lifecycle scheme: `fkst-goal` (replaces the visible `fkst-hosted:goal`), `fkst-session-<id>` (links the spawned session), `fkst-running`, and the three terminal labels `fkst-terminated` / `fkst-completed` / `fkst-failed`. Labels are updated read-then-replace (preserving unrelated labels, including `fkst-goal` and the session link) via a new best-effort `IssueApi::get_issue_labels` + the store's `update_labels` helper — a label write never fails or stalls a session. `SessionDoc` gains a persisted `terminal_cause` discriminator (`Option<TerminalCause>`, omitted while live) so a user-stop (`terminated`), a graceful engine completion (`completed`, an uncommanded clean exit 0 — no longer mis-recorded as `failed`), and an error (`failed`) are distinguishable both on the document and on the issue; `GET /api/v1/sessions/{id}` surfaces it. The label sync is wired into the in-process driver only; the worker-dispatch path's equivalent is deferred to #193. The hidden `marker.rs` server marker is unchanged and remains distinct from the visible `fkst-goal` label.
