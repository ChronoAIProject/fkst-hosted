# FKST Mission Control ‚Äî Design System (v2, locked)

The visual + interaction language for the FKST console. Locked from the shipped screen set **`designs/goal-board-20260611/`** ‚Äî `overview` ¬∑ `goals` ¬∑ `packages` ¬∑ `goal` ¬∑ `inbox` ¬∑ `runs` ¬∑ `settings` (all seven share one token system, one type stack, one logo) ‚Äî and the **v2 brand mark** (`fkst-brand-amber.html`: the amber **K-as-fork**). Reference implementation tokens are CSS custom properties; production uses Tailwind/shadcn mapped to these. **Every surface must read as one system.**

> Changes from v1: re-grounded from the old `mockups/04-overview.html` onto the seven production screens above; brand mark updated to v2 (the leading **F is set as normal type ‚Äî no mirror**; the standalone K-mark uses **butt caps + miter joins**, not round, so it reads as a letter, not a rounded icon); construction grid + a couple of verified component specs (topology graph, read/write tri-panel, condense-on-scroll topbar) folded in. Tokens are unchanged from what the screens actually ship.

## 1. Principles
- **Clean, modern, dark ‚Äî restraint over decoration.** Reference feel: Linear / Vercel / Railway. Flat canvas (no gradient, **no grid texture** ‚Äî the brand sheet's dot-grid is brand-surface only, never the app), hairline dividers, generous whitespace. **Type and space do the work, not borders, glows, or color.**
- **One accent.** Amber is the brand and the *only* brand accent. Everything else is neutral or a status semaphore. No second brand color.
- **No AI-slop.** Banned: glows / box-shadow halos, "marching-ant" animated dashes, emoji-as-icons, rainbow status palettes, a card-in-a-box for every element, big-number KPI tiles as decoration, decorative gradients/blobs, `system-ui` as the type, looped/orbiting logo treatments.
- **Hierarchy is deliberate.** Every screen has a clear 1st / 2nd / 3rd read (¬ß6). Neutral things recede (dim), important things advance (weight, size, a single colored rule).
- **Honesty (load-bearing).** Data is poll-derived (~5-min cron ticks), not live ‚Äî say so. State is re-derived from GitHub every poll, never cached. Never fake calm: a stale poll is surfaced; a source / the hosted service being **unreachable reads *unknown*, never "0"**. Show the deciding source's `as-of` (the screens carry a `state as of <n> ago` footer + a GitHub freshness chip).
- **Posture at the point of action.** The `WRITE: REAL` state is **not** global chrome ‚Äî it's a discreet posture chip and a contextual tag where a real write happens (the merge row / the Ship stage), reflecting the deployment's **global `FKST_GITHUB_WRITE`** ‚Äî **dry-run by default; REAL enables autonomous `gh pr merge` into the *integration branch* (never `main`; a rollup PR carries integration ‚Üí dev), completing in seconds**. Posture is a host/deployment fact, not a per-goal control.
- **Action grounding (load-bearing).** The FE fronts a **hosted** engine (a Rust backend running substrate+packages as ChronoAI cloud) that it **cannot command directly**. **Every action maps to exactly one of three real capabilities:** (1) a **GitHub mutation** ‚Äî open / comment / label / close / **create a new issue** ‚Äî via the GitHub API; (2) a **re-trigger / seed** of work through the hosted substrate (the package graph then processes it); (3) a **posture or topology** change on the hosted service (global `FKST_GITHUB_WRITE`; which packages load + how they compose). **Forbidden ‚Äî no FE/engine API exists:** mutating the redb delivery ledger / DLQ ("requeue"); *resuming* a terminal goal (`blocked` / `impl-failed` / `merged` have **no console out-edges** ‚Äî re-engagement is a **new** GitHub fact, never "reopen" / "retry"; the one program-defined exception is the engine's `fkst: rereview` operator command ‚Äî a **trusted-bot** PR comment, bot-issued only, so it is **not an FE action**); a *per-goal* pause (writes are global posture, autonomous, seconds-fast); or "connect a local host agent" (the engine is hosted, not local). State authority is the trusted-bot `state:v1` marker; `fkst-dev:*` labels are only self-heal hints.
- **Hosted v1 API grounding (fkst-hosted `develop`).** Capability (3) grounds in exactly this surface: `GET /health` ¬∑ `GET|POST|PUT|DELETE /api/v1/packages` (create `{name, files[], composed_deps[]}` with **409 on duplicate**, or update/delete packages via `/api/v1/packages/:name` - UI coming soon) ¬∑ `POST /api/v1/sessions` (`{package_name}`, **one live session per package, lease-fenced**) ¬∑ `GET /api/v1/sessions/:id` (status: `pending ‚Üí validating ‚Üí running ‚Üí stopping ‚Üí stopped / failed`) ¬∑ `POST /api/v1/sessions/:id/stop` (202; poll for truth). **"Apply changes / supervise restart" = stop session ‚Üí poll `stopped` ‚Üí create session.** A package loads **once** at session start and composes into **one static graph** ‚Äî there is **no hot-reload**, and the source tree is **read-only at runtime**. Create validates **structure only** (name `[A-Za-z0-9_-]+` ¬∑ ‚â•1 file ¬∑ ‚â§256 files ¬∑ ‚â§1 MiB/file ¬∑ ‚â§12 MiB total ¬∑ must contain `departments/*/main.lua` or `raisers/*.lua`); **conformance runs at session start** (a non-conforming root lands `failed` with the error on the session record). **v1 gaps the FE renders honestly (disabled control + note, not fictional success):** no posture endpoint (`FKST_GITHUB_WRITE` is deploy-time env today), no config-read endpoint (env panel reads "unknown"), no auth/NyxID integration yet (CORS wide open, backend TODO). Session progress is journaled to a GitHub repo per logical run (`run_key`) ‚Äî the durable session-history anchor for the Runs view.
- **Colorblind-safe.** Status is never hue-alone ‚Äî pair color with text + position (a label, a section, an icon-shape). WCAG AA contrast on the dark canvas.

| `--raise-2` | `oklch(23% 0.013 250)` | hover/active raised, chips, avatar |

**Accent + semaphore (used sparingly):**
| token | value | meaning ‚Äî strictly |
|---|---|---|
| `--amber` | `oklch(81% 0.15 80)` ‚âà `#F2B03C` | **brand only** ‚Äî logo K-node, active filter segment, enabled toggle, primary action, cross-package queue, focus ring. **Never a status.** |

- **Text on amber fills** = `oklch(22% 0.04 80)` ‚Äî a near-black warm ink, not pure black (so the fill reads as warm, not a sticker). Used on every primary/amber button and the active segment.
- **Tinted borders/fills** are built with `color-mix(in oklab, var(--accent) <n>%, var(--line)/transparent)` ‚Äî never a second opaque color token. (e.g. a danger chip = `color-mix(in oklab, var(--red) 38%, var(--line))` border over a `12%` wash.)

## 3. Typography
Loaded: `Space Grotesk` 500/600/700 ¬∑ `IBM Plex Sans` 400/450/500/600 ¬∑ `IBM Plex Mono` 400/500.
- **`--display` Space Grotesk** ‚Äî the wordmark, section headings (`h2` 16px / 600 / `+.01em`), modal titles (19px / 600 / `-.01em`), and big numerals (counts, vitals). Never body copy.
- **`--ui` IBM Plex Sans** ‚Äî all UI text. Body 14px / line-height 1.5. (Not `system-ui`.)
- **`--mono` IBM Plex Mono** ‚Äî IDs, SHAs, versions, timestamps, queue names, throughput, eyebrows, freshness. Always `font-variant-numeric: tabular-nums`.
- Scale: page numerals 27‚Äì34px (Space Grotesk 600‚Äì700, `letter-spacing:-.02em`); section heading 16px; modal title 19px; **eyebrow 11px mono `.18em` uppercase `--ghost`** (the structural label ‚Äî `WINDOW`, `DEPLOYMENT`, `SOURCES`); body 14px; nav/links 13.5px; meta/mono 10.5‚Äì12.5px. Body never below 12.5px; prefer 13‚Äì14. The tiniest mono labels (10‚Äì11px) are reserved for badges/queue chips only.

## 4. Logo / brand mark (v2)
The wordmark **FKST** where the **K *is* the mark** ‚Äî the in-product brand expression. No separate icon sits beside it in the chrome.
```html
<a class="logo" href="#">F<span class="kk">K<i class="knode"></i></span>ST</a>
```
- `.logo` = Space Grotesk **700**. The letters are drawn **exactly as the font sets them** ‚Äî `F`, `K`, `S`, `T` are never redrawn, synthesized, or mirrored (v1's mirrored-F is reverted). The *only* addition is the **amber consensus node**: an absolutely-positioned `.knode` dot at the K's junction with a `--bg` separation ring (`box-shadow:0 0 0 .05em var(--bg)`). At nav scale (‚âà19px) the node is `left:.04em; top:.36em; ~.205em` square ‚Äî **optically tuned per size**, larger at display scale; it is not a fixed em value, it's placed to sit on the K's fork.
- **Standalone K-mark** (favicon / icon-only / where the word can't go) ‚Äî the SVG `#mark`, a 96-unit K: stem + two mirrored diagonals meeting at the amber node. **Strokes are butt-cap + miter-join** (sharp, type-like ‚Äî *not* round) so it reads as a letter, not a generic rounded glyph. Construction: stem `x32 ¬∑ y16‚Üí82`, junction `32,46`, arm/leg splay `36¬∞ / 41¬∞`, stroke `11u`, node `r11.5u ¬∑ ring 3u`. Favicon variant `#mark-min` thickens the stroke to `13u` and the node to `r13u` so the fork survives below 24px.
- **The mark is static.** The brand sheet's looped/orbit/Command-K/figure-8 treatments are **experimental and explicitly not adopted** ‚Äî they imply real-time motion on poll-derived data and are out (¬ß8, ¬ß9).

## 5. Layout
- **Max width 1440px, centered** (`margin:0 auto`), `24px` side padding (`16px` under 480). **Hard cap ‚Äî never wider than 1440px**, even on huge displays. Capped + centered, never full-bleed; the side margins are intentional.
- **Condense-on-scroll topbar.** Header (`.hd`: logo ¬∑ nav ¬∑ GitHub freshness chip ¬∑ avatar) and the filter toolbar (`.toolbar`: primary action ¬∑ window/segment controls ¬∑ count summary) are wrapped in one sticky `.topbar`. On scroll they **collapse into a single row**: the eyebrow drops, the full controls hide, and the toolbar's current settings reappear as compact **icon + value "mini" chips**; a hairline section-pipe + the GitHub chip + avatar reflow inline. Use **hysteresis** to avoid flicker at the boundary ‚Äî condense once past `140px`, expand only back under `40px` (do **not** toggle on a single threshold). Settings opens from the **avatar**, never a nav item.
- **Responsive ‚Äî required, not desktop-only.** Every grid uses **`grid-template-columns: repeat(N, minmax(0,1fr))`** ‚Äî plain `1fr` is `minmax(auto,1fr)` and **refuses to shrink below its content**, overflowing the viewport. Pair with **`min-width:0`** on any flex/grid child that holds truncating/wrapping text (`overflow:hidden;text-overflow:ellipsis` and `overflow-wrap:anywhere` do nothing without it). **Verify `document.documentElement.scrollWidth <= window.innerWidth` at every breakpoint ‚Äî never eyeball it.** Breakpoints in the shipped screens: `1080` (4-col bands ‚Üí 2-col) ¬∑ `980` (multi-col panels ‚Üí single column; pipeline rail tightens) ¬∑ `780` (header wraps, breadcrumb + freshness hide, nav horizontal-scrolls, two-col rows stack) ¬∑ `600` (bands ‚Üí 1-col, dense rows go inline-wrap) ¬∑ `480` (`16px` padding, segmented controls may wrap; tap targets ‚â•44px).
- **The pipeline never wraps into 2 rows.** A pipeline reads as a pipeline only with one directional axis ‚Äî below full width it **scrolls horizontally, then stacks to a vertical column**, but never becomes a multi-row grid (that reads as a *table*). Goal titles wrap to 2 lines (`-webkit-line-clamp:2`), never truncate ‚Äî identifying the goal is the whole point. The **Board view** *is* a grid by nature, so its columns may horizontal-scroll when tight; that's fine. `clamp()` for fluid type/space where it helps.
- **Hairline dividers** (`--line`) separate regions and rows ‚Äî not bordered cards. The "list" pattern is a `--line`-background grid with `1px` gaps so each row is a `--raise` cell hairline-split from its neighbours (see `.plist`, `.levels`, `.rw`). Cards/panels (a real border + radius) only when the panel *is* the object (a goal card, a modal, a vitals panel).
- **Section eyebrows** (`DEPLOYMENT`, `SOURCES`, `NEEDS YOU`) and section headings structure the page; eyebrow = mono uppercase `--ghost`, heading = Space Grotesk 16px.
- Radii: 6‚Äì9px controls/chips, 10px row-cells/board-cards, 14px panels/lists, 16px modals.

## 6. Hierarchy contract (per screen, 1st ‚Üí 2nd ‚Üí 3rd)
- **Overview (home) ‚Äî two views, Pipeline or Board** (icon switch in the toolbar ‚Äî pipe glyph + board glyph; active = **amber stroke**, not amber fill): selecting any goal (rail row or board card) opens the **Issue modal** (¬ß7). **1** the canvas (hero, most space). **Pipeline view** = a single horizontal rail left‚Üíright where only the **4 active stages** **Design ¬∑ Build ¬∑ Review ¬∑ Ship** are columns goals live in (stage is **Design**, not "Consensus" ‚Äî consensus is the *mechanism*, used in both Design and Review); **Intake** (the cron faucet) and **Merged** (terminal, already counted) are slim **end-cap nodes**, not full stages; titles wrap to 2 lines; trouble stages advance via a colored top rule (Review `--gold` / Ship `--red`), neutrals recede. **Board view** = the same goals as **cards in kanban columns by stage** (the "show me everything" view; subtle `--raise` cards, hairline column dividers, colored column accents for Review/Ship/Merged). **2** the **vitals panel** (windowed at-a-glance). **3** **Needs you** (subordinate action list, under both views).
- **Goals (manage list ‚Äî Runs folded in as an Issues/Activity view):** **1** the goal list (state pill ¬∑ `#id` ¬∑ title ¬∑ stage ¬∑ age) **2** filters/window **3** activity / raw markers.
- **Goal page (depth):** **1** current state + the one decision needed **2** the lifecycle timeline (trust + as-of) **3** diagnostics (runs, raw markers).
- **Packages (manage) ‚Äî anchored on the Company ¬∑ Department ¬∑ Person model:** an intro states the behavior layer in one breath (Lua on the hosted engine, package store, session scans once ‚Üí one static graph, changes apply on restart, source read-only). **1** the **loaded-package list** (flat/composed badges, `composed.deps` chips, conformance ‚úì, per-package enable toggle, the active goal-engine badge) + **Apply changes ¬∑ stop & restart session**. **2** the **composed graph / topology** (a SOURCES band of cron raisers, then Departments in pipeline order as `consumes ‚Üí produces` rows; nodes = departments, edges = queues, one `codex exec` per event, cross-package queues amber, terminal queues green). **3** the **read/write boundary** tri-panel (source tree read-only ¬∑ FE manages set/topology/posture via restart ¬∑ business writes ‚Üí GitHub under REAL).
- **Settings (account ‚Äî opened from the avatar):** **1** account + **connected repos** + **poll interval** **2** **write posture & safety** (the gated REAL elevation; global `FKST_GITHUB_WRITE`) + connections (NyxID, hosted backend) **3** deployment knobs + danger zone (**delete account**). v1 gaps (no posture endpoint, no config read, no auth) render as disabled controls + an honest note ‚Äî never fictional success.
- **Inbox** ‚Äî deferred / hidden in the shipped nav for now (complex). Spec retained for when it returns; do not add it to the primary nav until built.

## 7. Components
- **Top bar** ‚Äî **brand + nav + freshness + avatar**, no breadcrumb. The FKST wordmark; nav (`Overview ¬∑ Goals ¬∑ Packages`, active = `--raise` fill, hover ‚Üí `--raise-2`); a right-aligned **GitHub status chip** (octocat icon + a `--green` health dot, title carries the as-of) and the round **avatar** (initials; the account/settings entry). Condenses on scroll (¬ß5).
- **Window control** ‚Äî segmented (`Live ¬∑ 1h ¬∑ 24h ¬∑ 7d ¬∑ 30d`); inactive segment hover = `--dim` + faint wash; **active segment fills `--amber`** with `oklch(22% 0.04 80)` text (the one amber moment in the control). Drives every windowed count/rate.
- **Vitals panel** ‚Äî one contained `--raise` panel, metrics split by hairline `--line` rules; big Space Grotesk numeral + `--faint` label; the danger metric (dead-ended) in `--red`, success (merged) in `--green`. No glow, no decorative KPI tiles.
- **Pipeline stage** ‚Äî one of the 4 active columns on a single horizontal rail (Intake/Merged are slim end-cap nodes, *not* stages), split from neighbours by a vertical `border-left` hairline, **open top/bottom with one continuous conduit line** through the nodes so it reads as a **pipe, not a table**; Space Grotesk name + large count; windowed `in/out`; a clean goal list (dot + mono `#id` + 2-line title + age). Trouble = a 2px inset top rule (`--gold`/`--red`) + one quiet tag. **No glow.**
- **Board (kanban) ‚Äî Overview's second view** ‚Äî columns are the goal-bearing stages; each column = header (name + count, colored bottom-rule for Review/Ship/Merged) over a stack of **goal cards**. A card is a subtle `--raise` panel (`--line` border, 10px radius ‚Äî the card *is* the goal object, so a panel is earned): mono `#id` + a small state badge, a 2-line title, mono meta (`repo ¬∑ PR ¬∑ age`). Hover ‚Üí `--raise-2`. **No glow, no drag-shadow.**
- **Issue modal** ‚Äî selecting a goal anywhere opens one detail card in a modal: dim backdrop (`color-mix(in oklab,#000 58%,transparent)`), `--raise` sheet, `--line-2` border, a **soft seat-shadow** (`0 24px 60px -22px rgba(0,0,0,.6)` ‚Äî depth, *not* a glow), Esc / backdrop / √ó to close. Carries marker-derived info: a **state pill** + `#id` + title; a meta line (repo ¬∑ stage ¬∑ PR ¬∑ CI ¬∑ age); the **lifecycle timeline** of trusted `state:v1` markers (current node = amber); the **consensus** summary (CONVERGE in Design / GATE in Review); the **merge gate** (CI ¬∑ review-approve ¬∑ head-bound ¬∑ mergeable ‚Üí REAL ‚Üí integration, or "not at the gate yet"); a **provenance** line (markers are fact, labels are hints); and **grounded actions only** ‚Äî Open issue ‚Üó / Open PR ‚Üó / Full goal page ‚Üí, plus context (Ship ‚Üí *Global write ‚Üí DRY-RUN*; terminal ‚Üí *New issue from this*). Never a per-goal engine control.
- **List / row pattern** (`.plist`, `.levels`, `.rw`) ‚Äî a `--line`-background grid, `1px` gaps, each cell `--raise`; row hover brightens toward `--raise-2`. The default way to show a set without bordering every item.
- **Package row** ‚Äî mono name + a **flat/composed badge** (composed = amber-tinted) + optional **active goal-engine** badge (green-tinted); a plain-language role line; mono meta (`N departments ¬∑ N raisers ¬∑ conformance ‚úì`); `composed.deps` as `--raise-2` chips; a right-side **enable toggle** (the system switch ‚Äî `--line-2` track off, amber track+knob on) + a `View source ‚Üó` link. Disabled package ‚Üí `opacity:.62`.
- **Topology graph** ‚Äî not a node-and-wire canvas; a legible **adjacency list**. A SOURCES band (raiser cards: name ¬∑ `cron 5m` with an amber dot ¬∑ the tick queue) above Departments in pipeline order, each a `consumes ‚Üí produces` row with queue chips. Cross-package queue chip = **amber** (`pkg.queue`), terminal queue = **green**, a per-department `codex ‚úì / no codex` tag (gold when it runs an exec). Collapsed nodes are disclosed honestly in a footnote ("21 departments total; 4 collapsed for legibility ‚Äî ‚Ä¶").
- **Posture chip** ‚Äî discreet, *contextual* (point of action), never a global banner: DRY-RUN = `--faint` on `--raise`; REAL = `--red` text + `color-mix` red border + a red dot. The read/write boundary panel's "business writes" column carries the same `REAL posture required` tag.
- **+ New goal / + Add package (trigger)** ‚Äî a primary **amber** button in the toolbar opens a modal. **New goal**: repo + title + description ‚Üí **creates a GitHub issue + `fkst-dev:enabled`** (capability #1/#2) ‚Äî honest copy that this is a *real GitHub write via NyxID* (works in DRY-RUN; not an engine write); the next ~5-min poll + `intake_judge` picks it up ‚Üí Design. The modal's **Packages** section toggles which deployment packages process it (honest note: **deployment-wide, applies on restart** ‚Äî not per-goal). **Add package**: `{name, files[], composed_deps[]}` ‚Üí `POST /api/v1/packages`, with the full create-only / 409 / validation / conformance-at-start contract stated inline.
- **Buttons** ‚Äî secondary (default): `--raise`/`--raise-2` fill, `--line-2` border, `--dim` text; hover ‚Üí `--fg` text + `--faint` border (neutral brighten, **not** amber). Primary: amber fill, `oklch(22% 0.04 80)` text, hover `brightness(1.06)`. Danger: `--red` outline, hover faint `--red` wash. Dashed-border "add" affordance: `--line-2` dashed ‚Üí solid `--faint` on hover.

### 7.1 Interactive states (default / hover / active / focus ‚Äî consistent everywhere)
**Amber is reserved** for the *active filter segment*, the *enabled toggle*, *primary/brand* actions, and the *focus ring* ‚Äî **not** every hover. Default hovers are a neutral brighten.
- **Nav link** ‚Äî default `--faint`; **hover** `--dim` + faint `--raise` wash; **active** `--fg` + `--raise` fill (hover ‚Üí `--raise-2`).
- **Secondary button** ‚Äî hover brightens to `--fg` text + `--faint` border (neutral); active slightly darker.
- **Segmented control** ‚Äî inactive hover = `--dim` + faint wash; **active = amber fill**.
- **Toggle / switch** ‚Äî off = `--line-2` track + `--faint` knob; **on = amber track + amber knob** (+ amber label).
- **Clickable surface (stage column, row, card)** ‚Äî hover = faint `--raise`/`--raise-2` wash + `cursor:pointer`; whole-surface hover signals "drill in."
- **Focus (keyboard)** ‚Äî every interactive element gets `:focus-visible{outline:2px solid var(--amber);outline-offset:2px}`. Non-negotiable for a11y; the only place amber appears on a non-active element.
- **Transitions** ‚Äî `.12s ease` on background/color/border only. No transform/scale bounce.

## 8. Motion
Minimal. **No looping/marching/orbiting animations** (they imply real-time on poll-derived data ‚Äî and rule the brand sheet's experimental looped marks out of the product). Subtle, brief transitions on interaction only; `prefers-reduced-motion` respected everywhere. A hero logo draw-in is acceptable on a splash/marketing surface only, never in the console chrome.

## 9. Don't (the anti-slop checklist)
No glows ¬∑ no animated dashed conduits ¬∑ no orbiting/looped logo ¬∑ no emoji icons ¬∑ no rainbow status ¬∑ no card-mosaic (box around everything) ¬∑ no decorative gradients/blobs ¬∑ no grid texture in-app ¬∑ no centered-everything ¬∑ no `system-ui` type ¬∑ no faking "live" or "0" (unreachable = *unknown*) ¬∑ no amber-as-status ¬∑ no per-goal engine control ¬∑ no plain `1fr` grids (use `minmax(0,1fr)` + `min-width:0`).
